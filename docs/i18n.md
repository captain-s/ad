# 国际化

AD Maven 提供了完整的国际化支持，基于 i18next 实现，支持多语言切换和翻译管理。

## 国际化配置

国际化配置位于 `src/locales` 目录下，主要包含以下文件：

- `i18n.ts`：i18next 配置
- `use-locale.ts`：国际化钩子
- `lang/`：语言资源文件
  - `en_US/`：英文资源
  - `zh_CN/`：中文资源

## 语言资源文件

语言资源文件采用 JSON 格式，按功能模块组织：

### 英文资源

```json
// src/locales/lang/en_US/common.json
{
  "hello": "Hello",
  "welcome": "Welcome to AD Maven",
  "login": "Login",
  "register": "Register",
  "logout": "Logout"
}
```

### 中文资源

```json
// src/locales/lang/zh_CN/common.json
{
  "hello": "你好",
  "welcome": "欢迎使用 AD Maven",
  "login": "登录",
  "register": "注册",
  "logout": "退出"
}
```

## 国际化使用

### 基本用法

使用 `useTranslation` 钩子进行翻译：

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t("welcome")}</h1>
      <button>{t("login")}</button>
    </div>
  );
}
```

### 带参数的翻译

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <p>{t("greeting", { name: "John" })}</p>
    </div>
  );
}
```

对应的语言资源：

```json
// src/locales/lang/en_US/common.json
{
  "greeting": "Hello, {{name}}!"
}

// src/locales/lang/zh_CN/common.json
{
  "greeting": "你好，{{name}}！"
}
```

### 语言切换

使用 `useLocale` 钩子切换语言：

```tsx
import useLocale from "@/locales/use-locale";

function LanguageSelector() {
  const { language, changeLanguage } = useLocale();

  return (
    <select
      value={language.key}
      onChange={(e) => changeLanguage(e.target.value)}
    >
      <option value="en_US">English</option>
      <option value="zh_CN">中文</option>
    </select>
  );
}
```

## 国际化最佳实践

1. **统一管理**：将所有翻译资源统一管理在 `locales/lang` 目录下
2. **模块划分**：按功能模块划分翻译资源，便于管理
3. **键名规范**：使用语义化的键名，如 `button.login`、`message.error` 等
4. **默认语言**：设置默认语言，确保所有场景都有翻译
5. **动态内容**：使用参数化翻译处理动态内容
6. **测试**：确保所有翻译都能正确显示

## 添加新语言

要添加新语言，按照以下步骤操作：

1. 在 `src/locales/lang` 目录下创建新的语言目录，如 `ja_JP`
2. 在新目录下创建语言资源文件，如 `common.json`、`sys.json` 等
3. 在 `src/locales/i18n.ts` 文件中添加新语言的配置
4. 在语言选择器中添加新语言选项

## 常见问题

### 翻译不生效

- 检查语言资源文件是否正确
- 检查键名是否匹配
- 检查语言是否正确设置

### 动态内容翻译

- 使用参数化翻译处理动态内容
- 避免在翻译键中硬编码动态内容

### 性能问题

- 合理使用翻译缓存
- 避免在渲染过程中频繁调用翻译函数
- 考虑使用 `Trans` 组件处理复杂的翻译内容
